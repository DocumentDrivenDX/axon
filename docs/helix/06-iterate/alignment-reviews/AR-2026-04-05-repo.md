## Intent Summary

- **Vision**: Axon is the central nervous system for agentic applications — a cloud-native, auditable, schema-first transactional data store. Every agentic application should have an Axon for structured state.
- **Requirements**: V1 P0 scope covers entity model, link model, collections, schema engine, audit log, CRUD, ACID transactions, OCC, API surface, embedded mode, CLI. P1 adds schema evolution, change feeds, auth, graph traversal, server mode. P2 adds local-first sync, workflow primitives.
- **Features / Stories**: 10 feature specs (FEAT-001 through FEAT-010) with ~126 acceptance criteria. P0: FEAT-001 (Collections), FEAT-002 (Schema), FEAT-003 (Audit), FEAT-004 (Entity Ops), FEAT-005 (API), FEAT-007 (Entity-Graph Model), FEAT-008 (Transactions). P1: FEAT-006 (Bead Adapter), FEAT-009 (Graph Traversal). P2: FEAT-010 (Workflow State Machines).
- **Architecture / ADRs**: Rust implementation (ADR-001). JSON Schema Draft 2020-12 + Axon link-type definitions (ADR-002). SQLite (embedded) + PostgreSQL (server) with application-layer audit (ADR-003). StorageAdapter trait as abstraction layer.
- **Technical Design**: No formal solution designs or technical designs exist. Architecture decisions captured in ADRs. Implementation follows StorageAdapter trait pattern with AxonHandler<S> generic handler.
- **Test Plans**: FoundationDB-style 6-layer test architecture: L1 correctness invariants (8 INV), L2 business scenarios (10 SCN), L3 property-based (5 PROP), L4 backend conformance (parameterized), L5 performance benchmarks (10 BM), L6 API contract tests.
- **Implementation Plans**: No formal implementation plan document exists. Execution tracked via tracker beads.

## Implementation Map

- **Topology**: Cargo workspace with 8 crates — `axon-core` (types/errors), `axon-schema` (validation), `axon-audit` (immutable log), `axon-storage` (trait + SQLite/Memory), `axon-api` (handler + request/response), `axon-cli` (binary), `axon-server` (gRPC + HTTP gateway), `axon-sim` (deterministic simulation).
- **Entry Points**: CLI binary (`crates/axon-cli/src/main.rs`), HTTP gateway (`crates/axon-server/src/gateway.rs`), gRPC service (`crates/axon-server/src/service.rs`). No server binary entry point (`main.rs`) — `axon-server` is a library.
- **Test Surfaces**: 79 tests passing (0 failures). L1 invariants (8/8), L2 scenarios (10/10), L3 property (5/5), L4 conformance (SQLite + Memory), L5 benchmarks (10/10). L6 API contract tests not formally implemented. Server crate has 0 tests.
- **Unplanned Areas**: Filter depth limit (defensive security measure, not documented in specs). Reverse-index optimization for inbound links (`__axon_links_rev__` collection). `register_schema()` deprecated method.
- **Dead/Orphaned**: `Cardinality` enum defined in schema but cardinality enforcement not implemented (only target_collection and metadata_schema are enforced).
