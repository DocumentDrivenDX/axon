## 5. Implementation Map

- **Topology**: 13-crate workspace: `axon-core`, `axon-schema`, `axon-audit`, `axon-storage`, `axon-api`, `axon-render`, `axon-graphql`, `axon-mcp`, `axon-server`, `axon-sim`, `axon-cli`, `axon-config` (new), plus admin UI under `ui/`, TypeScript SDK under `sdk/typescript/`, website under `website/`.
- **Entry points**: `axon` binary (unified): `axon serve` (HTTP/gRPC/MCP), `axon mcp` (stdio), `axon doctor`, `axon install`, CLI commands. Server exposes HTTP gateway, optional gRPC, control plane, GraphQL, MCP endpoints.
- **Public interfaces**: HTTP gateway exposes entity, collection, schema, audit, transaction, rollback, template, tenant (control plane), GraphQL (`/graphql`, `/graphql/ws`), and UI routes. gRPC exposes CRUD/query/schema/database APIs. MCP HTTP and stdio transports active. Auth is enforced via Tailscale whois middleware with role boundary checks on write operations.
- **Multi-tenancy**: `TenantRouter` wired into gateway via `X-Axon-Database` middleware; SQLite per-tenant isolation live; PostgreSQL provisioning pending.
- **Test surfaces**: 464+ Rust unit/integration tests pass. 42 RBAC integration tests. 78 Playwright E2E tests. UI bun tests in CI. All format and machete gates pass. Clippy and cargo deny gates currently failing.
- **Unplanned or incomplete**: PG per-tenant provisioning (in progress), HTTP direction=reverse contract test, clippy/deny CI failures.
