## 5. Implementation Map

- Topology: The workspace contains `axon-core`, `axon-schema`, `axon-audit`, `axon-storage`, `axon-api`, `axon-cli`, `axon-sim`, `axon-server`, `axon-graphql`, `axon-mcp`, and `axon-render` under `crates/`. The admin UI lives under `ui/`, and the TypeScript SDK lives under `sdk/typescript/`.
- Entry points: `crates/axon-server/src/main.rs` runs HTTP, gRPC, and MCP surfaces and can serve built UI assets from `--ui-dir` under `/ui`. `crates/axon-cli/src/main.rs` is the embedded operator surface.
- Public interfaces: `crates/axon-server/src/gateway.rs` exposes HTTP entity, collection, schema, audit, transaction, rollback, and UI routes plus MCP HTTP routes, but no `/graphql` or `/graphql/ws` routes. `crates/axon-server/src/service.rs` exposes gRPC CRUD/query/schema/database APIs. `ui/src/routes/` provides collection, schema, and audit pages over the HTTP gateway.
- Test surfaces: `cargo check`, `cargo test`, strict `cargo clippy`, `cargo fmt --check`, `cargo deny check`, `cargo machete crates`, `bun run typecheck`, `bun run lint`, `bun test`, and `bun run build` all passed on this pass.
- Unplanned / incomplete areas: GraphQL still exists mainly as an internal crate with tests rather than a public server surface; FEAT-012 role enforcement is still missing on HTTP and gRPC handlers; schema evolution is only partially surfaced publicly; point-in-time collection and transaction rollback are still missing; the UI still lacks pre-commit schema preview and broad route-level/browser-level regression coverage; FEAT-024 has no generated substrate surface; FEAT-025 has no implementation surface.
