## 5. Implementation Map

- Topology: The Rust workspace currently contains `axon-core`, `axon-schema`, `axon-audit`, `axon-storage`, `axon-api`, `axon-cli`, `axon-sim`, `axon-server`, `axon-graphql`, `axon-mcp`, and `axon-render` (`Cargo.toml:1-14`). The admin UI lives under `ui/`.
- Entry Points: `crates/axon-cli/src/main.rs` is the embedded operator surface; `crates/axon-server/src/main.rs` runs HTTP, gRPC, and MCP surfaces; `ui/` is a SvelteKit static app built with Bun.
- Public Interfaces: HTTP and request-scoped current-database routing live in `crates/axon-server/src/gateway.rs`; MCP HTTP+SSE lives in `crates/axon-server/src/mcp_http.rs`; collection, entity, audit, and schema UI flows live in `ui/src/routes/`.
- Test Surfaces: `cargo test` passes across handler, storage, server, gRPC, MCP, backend parity, business-scenario, Postgres parity, and simulation suites. Strict clippy, fmt, deny, and machete also passed. `bun run typecheck`, `bun run lint`, `bun test`, and `bun run build` all pass after `bun install`.
- Unplanned / incomplete areas: schema evolution is still only partially surfaced publicly; `revalidate` exists only as an internal handler path; point-in-time collection rollback and transaction rollback are still missing; the UI still lacks pre-commit schema preview and route-level regression coverage; the implementation-plan lane and digest hygiene remain missing.
