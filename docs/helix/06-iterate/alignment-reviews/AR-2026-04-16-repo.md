## 3. Intent Summary

- **Vision**: Axon is an auditable, schema-first OLTP store for human and agentic workflows, exposed through JSON, GraphQL, and MCP with operator-friendly browser and CLI surfaces. Unchanged since prior review.
- **Requirements**: PRD P0 centers on CRUD, links, OCC, transactions, audit, API, embedded mode, and CLI — all SATISFIED. P1 adds auth/RBAC, GraphQL, MCP, schema evolution, admin UI, rollback, templates, control plane, multi-tenancy, and guardrails — all now substantively implemented. ADR-018 introduced a major architectural change: tenant as global account boundary with M:N users, JWT credentials, and pure path-based wire protocol. P2 includes guardrails extensions, application substrate, git mirror, and workflow primitives — guardrails partially SATISFIED (rate limiting + scope constraints), git mirror draft-only.
- **Features / Stories**: All P0 and P1 features are now at ALIGNED or SATISFIED. The ADR-018 migration is the single largest delta: path-based routing, JWT auth pipeline, tenant membership CRUD, credential lifecycle, and the removal of all legacy routes. This brought FEAT-012 from Partial to V5 In Evolution, FEAT-014 from basic namespaces to full four-level hierarchy.
- **Architecture / ADRs**: ADR-018 is the newest and most consequential decision. All 18 ADRs remain consistent with implementation. ADR-005 (Tailscale auth) is fully implemented via direct Unix socket HTTP (no `tailscale_localapi` crate). ADR-018 cleanly supersedes the prior `efe4aa1` simplification.

## 5. Implementation Map

- **Topology**: 13-crate workspace: `axon-core`, `axon-schema`, `axon-audit`, `axon-storage`, `axon-api`, `axon-render`, `axon-graphql`, `axon-mcp`, `axon-server`, `axon-sim`, `axon-cli`, `axon-config`, `axon-control-plane`. Admin UI under `ui/`. TypeScript SDK under `sdk/typescript/`.
- **Entry points**: `axon` binary (unified): `axon serve` (HTTP/gRPC/MCP), `axon mcp` (stdio), `axon doctor`, `axon install`, CLI commands. Server exposes HTTP gateway with pure path-based routing per ADR-018.
- **Public interfaces**: HTTP gateway: `/tenants/{t}/databases/{d}/...` for all data-plane routes (entities, collections, schemas, audit, transactions, rollback, templates, lifecycle). Control plane: `/control/tenants`, `/control/users`, `/control/tenants/{id}/members`, `/control/tenants/{id}/credentials`, `/control/tenants/{id}/databases`, `/control/tenants/{id}/retention`. GraphQL: `/tenants/{t}/databases/{d}/graphql` + WebSocket. MCP: `/mcp` + stdio. UI: `/ui/...`.
- **Auth**: JWT credentials verified via 9-step pipeline in `auth_pipeline.rs`. Tailscale whois as identity provider via Unix socket. `--no-auth` and `--guest-role` modes for development. RBAC role checks on all write paths.
- **Multi-tenancy**: `PathRouter` extracts `(tenant, database)` from URL. `DatabaseRouter` + `DatabaseAdapterFactory` resolve tenant→database→storage adapter. Per-tenant database isolation (SQLite fully implemented, PostgreSQL provisioning now closed).
- **Test surfaces**: 1,039 Rust unit/integration tests pass. 42+ RBAC integration tests. 6 Playwright E2E test specs. All 8 DST correctness invariants verified. UI tests via bun. All format and machete gates pass. Clippy and cargo deny gates currently failing.
- **Unplanned or incomplete**: sqlx migration (in progress), semantic validation hooks (deferred), Kafka CDC (deferred), L2/L5 test layers (not started).
