## 5. Implementation Map

- Topology: The Rust workspace contains `axon-core`, `axon-schema`, `axon-audit`, `axon-storage`, `axon-api`, `axon-cli`, `axon-sim`, `axon-server`, `axon-graphql`, `axon-mcp`, and `axon-render` ([`Cargo.toml`](/home/erik/Projects/axon/Cargo.toml#L1)). The admin UI lives under `ui/`.
- Entry Points: [`main.rs`](/home/erik/Projects/axon/crates/axon-server/src/main.rs#L40) runs HTTP, gRPC, and MCP surfaces and can serve built UI assets from `--ui-dir`. [`main.rs`](/home/erik/Projects/axon/crates/axon-cli/src/main.rs) is the embedded operator surface.
- Public Interfaces: [`gateway.rs`](/home/erik/Projects/axon/crates/axon-server/src/gateway.rs#L1463) exposes HTTP entity, collection, schema, audit, and transaction routes, but no `/graphql` or `/graphql/ws` routes. [`mcp_http.rs`](/home/erik/Projects/axon/crates/axon-server/src/mcp_http.rs) implements MCP HTTP+SSE. UI flows live in `ui/src/routes/`.
- Test Surfaces: `cargo check`, `cargo test`, strict `cargo clippy`, `cargo fmt --check`, `cargo deny check`, `cargo machete crates`, `bun run typecheck`, `bun run lint`, `bun test`, and `bun run build` all passed on this pass.
- Unplanned / incomplete areas: GraphQL still exists mainly as an internal crate and open build queue, not a public server surface; schema evolution is only partially surfaced publicly; point-in-time collection rollback and transaction rollback are still missing; the UI still lacks pre-commit schema preview and route-level regression coverage; FEAT-024 has no generated substrate surface; the implementation-plan lane and digest hygiene remain missing.
