### Summary

The plan has converged on its hardest point: the `default-features = false` + formats-on + three-deps-only triad is internally consistent because jsonschema 0.28.3's format validators are non-optional, so the empty-features config retains them while dropping `reqwest`. Implementation can largely proceed. The blocking-adjacent gap is that the same dependency change *also* disables external `$ref` resolution for both entity and link validation — a workspace-wide behavior change the closure evidence does not name, contradicting Decision #5's "not tightened" framing and the bead's own requirement to name behavior changes. Pin that (plus the derive set for moved types and the `EsfCoreDocument`↔`EsfDocument` relationship) before closing.
