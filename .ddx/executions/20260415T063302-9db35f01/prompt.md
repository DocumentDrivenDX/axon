    <description>
--no-auth mode per-URL in-memory namespace. When the server runs with --no-auth, every request synthesizes an anonymous admin claim, and the storage materializes a per-URL (tenant, database) namespace on first touch. No persistent bootstrap, no user rows, no tenant rows written to disk.&#xA;&#xA;## Scope&#xA;&#xA;### 1. No-auth synthetic claim helper&#xA;&#xA;Add to crates/axon-server/src/auth_pipeline.rs (or a new module crates/axon-server/src/no_auth.rs):&#xA;&#xA;    /// Synthesize a ResolvedIdentity for --no-auth mode.&#xA;    ///&#xA;    /// The claim carries the anonymous user UUID (a deterministic zero UUID),&#xA;    /// the tenant and database names extracted from the URL path, and an&#xA;    /// admin grant on the database.&#xA;    pub fn synthesize_no_auth_identity(&#xA;        tenant: &amp;str,&#xA;        database: &amp;str,&#xA;    ) -&gt; ResolvedIdentity {&#xA;        ResolvedIdentity {&#xA;            user_id: UserId::nil(),  // deterministic anonymous UUID&#xA;            tenant_id: TenantId::from_name(tenant),  // hashable mapping&#xA;            grants: Grants {&#xA;                databases: vec![GrantedDatabase {&#xA;                    name: database.to_string(),&#xA;                    ops: vec![Op::Read, Op::Write, Op::Admin],&#xA;                }],&#xA;            },&#xA;        }&#xA;    }&#xA;&#xA;TenantId::from_name is a deterministic hash-to-UUID (use uuid::Uuid::new_v5 with a fixed namespace). This keeps the ID stable across calls for the same name without needing a DB lookup.&#xA;&#xA;### 2. Middleware/layer glue&#xA;&#xA;Add a new axum middleware in crates/axon-server/src/no_auth.rs (NEW):&#xA;&#xA;    pub async fn no_auth_layer(&#xA;        resolved_path: Option&lt;Extension&lt;ResolvedPath&gt;&gt;,&#xA;        mut req: Request,&#xA;        next: Next,&#xA;    ) -&gt; Response {&#xA;        if let Some(Extension(path)) = resolved_path {&#xA;            let identity = synthesize_no_auth_identity(&amp;path.tenant, &amp;path.database);&#xA;            req.extensions_mut().insert(identity);&#xA;        }&#xA;        next.run(req).await&#xA;    }&#xA;&#xA;Uses D1&#39;s ResolvedPath extension as the upstream input. This means the path_router_layer from D1 must run BEFORE no_auth_layer — add a doc comment saying so.&#xA;&#xA;### 3. In-memory namespace behavior&#xA;&#xA;The storage adapter layer doesn&#39;t need to change — the existing memory adapter already materializes (tenant, database) key spaces on demand. What this bead adds is the PROOF that --no-auth reaches the right adapter + namespace.&#xA;&#xA;Integration test proves:&#xA;- Writing to /tenants/a/databases/x/... and reading back succeeds (same namespace)&#xA;- Writing to /tenants/a/databases/x/... and reading /tenants/b/databases/y/... returns 404 (different namespace)&#xA;- After server restart (on an in-memory adapter), everything is gone&#xA;&#xA;### 4. Hit /tenants/acme/databases/orders/ping and assert the handler sees ResolvedIdentity with admin grants on &#34;orders&#34;.&#xA;&#xA;#### middleware_skips_when_path_not_data_plane&#xA;Hit /health — no ResolvedPath extracted, no ResolvedIdentity installed, handler sees no identity.&#xA;&#xA;## Out of scope&#xA;&#xA;- Wiring no_auth_layer into the real router at serve.rs — that&#39;s D3&#39;s job.&#xA;- The storage adapter per-URL namespace materialization — already works for the memory adapter.&#xA;- --no-auth + Tailscale fallback — B3 federation handles the Tailscale path; --no-auth is the &#34;no auth at all&#34; path.&#xA;&#xA;## Context&#xA;&#xA;- D1 landed path_router_layer + ResolvedPath&#xA;- B1 landed ResolvedIdentity (in axon-core)&#xA;- A2: Grants, GrantedDatabase, Op, TenantId, UserId&#xA;&#xA;## Files to touch&#xA;&#xA;- crates/axon-server/src/no_auth.rs — NEW (both synthesize fn + layer)&#xA;- crates/axon-server/src/lib.rs — pub mod no_auth; re-export&#xA;- crates/axon-core/src/auth.rs — add TenantId::from_name helper (uses uuid::Uuid::new_v5 with a fixed namespace)&#xA;- crates/axon-server/tests/no_auth_test.rs — NEW&#xA;&#xA;## Files NOT to touch&#xA;&#xA;- crates/axon-server/src/serve.rs — router wiring is D3&#xA;- crates/axon-server/src/auth.rs — legacy Tailscale&#xA;- crates/axon-server/src/auth_pipeline.rs — B1 is canonical, no_auth is a peer module&#xA;&#xA;## Verification&#xA;&#xA;- cargo test -p axon-server --test no_auth_test (5 tests pass)&#xA;- cargo test -p axon-core (TenantId::from_name unit test)&#xA;- cargo test --workspace (all prior green)&#xA;- cargo clippy --workspace -- -D warnings clean
