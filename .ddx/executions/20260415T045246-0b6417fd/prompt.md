    <title>path-router: axum layer extracts (tenant, database) from URL</title>
    <description>
Add a reusable axum middleware layer that extracts (tenant, database) from the URL path /tenants/{tenant}/databases/{database}/..., validates the identifiers, and installs an extension the downstream handlers and the JWT pipeline (B1) can read. This bead does NOT wire the middleware into the main router — that is D3&#39;s job. This bead builds the primitive + its tests. The primitive is a pure function + a thin axum layer wrapper.&#xA;&#xA;## Scope — exactly what to add&#xA;&#xA;### 1. New module: crates/axon-server/src/path_router.rs&#xA;&#xA;Three pieces:&#xA;&#xA;(a) A pure extractor function:&#xA;&#xA;    pub fn extract_tenant_database(path: &amp;str) -&gt; Option&lt;(String, String)&gt; {&#xA;        // Match /tenants/{tenant}/databases/{database}(/...)?&#xA;        // Return (tenant, database) for data-plane paths&#xA;        // Return None for unprefixed paths, /health, /ui/*, /control/*&#xA;    }&#xA;&#xA;This MUST be pure (takes &amp;str, returns Option&lt;(String, String)&gt;), MUST validate both segments against the identifier rule:&#xA;- 1-63 characters&#xA;- ASCII only: [a-zA-Z0-9_-]&#xA;- Cannot start with a digit&#xA;- Cannot be empty&#xA;&#xA;Reserved prefixes that bypass extraction (return None cleanly, no error):&#xA;- /health&#xA;- /metrics (for future Prometheus scrape)&#xA;- /ui (and /ui/*)&#xA;- /control (and /control/*)&#xA;- /favicon.ico&#xA;- /robots.txt&#xA;&#xA;Any other path that does not start with /tenants/ returns None. Paths that start with /tenants/ but fail the identifier rule ALSO return None — invalid identifiers are treated as &#39;not a data-plane route&#39; and fall through, so the caller can emit a 404. (Do not emit 400 here; the middleware layer decides whether to 404, and 400 on bad identifiers is too chatty for the common &#39;wrong URL&#39; case.)&#xA;&#xA;(b) A ResolvedPath struct that goes into the request extension:&#xA;&#xA;    #[derive(Debug, Clone, PartialEq, Eq)]&#xA;    pub struct ResolvedPath {&#xA;        pub tenant: String,&#xA;        pub database: String,&#xA;    }&#xA;&#xA;(c) The axum middleware layer as an async function:&#xA;&#xA;    pub async fn path_router_layer(&#xA;        mut req: axum::http::Request&lt;axum::body::Body&gt;,&#xA;        next: axum::middleware::Next,&#xA;    ) -&gt; axum::response::Response {&#xA;        let path = req.uri().path().to_string();&#xA;        if let Some((tenant, database)) = extract_tenant_database(&amp;path) {&#xA;            req.extensions_mut().insert(ResolvedPath { tenant, database });&#xA;        }&#xA;        next.run(req).await&#xA;    }&#xA;&#xA;The layer ALWAYS calls next — it does NOT decide 404s. Routing decisions (e.g., &#39;unrecognized /tenants/* path&#39;) are the router&#39;s job at the route table level. This is deliberate: the middleware&#39;s only job is to populate the extension when it can, and to leave it unpopulated when it cannot.&#xA;&#xA;### 2. Re-export from crates/axon-server/src/lib.rs&#xA;&#xA;pub mod path_router; and re-export path_router::{extract_tenant_database, path_router_layer, ResolvedPath}.&#xA;&#xA;### 3. Tests: crates/axon-server/tests/path_router_test.rs&#xA;&#xA;Unit tests on the pure extractor (NOT the middleware yet — that&#39;s easier to unit-test as a pure function):&#xA;&#xA;#### Happy path&#xA;- extract_tenant_database(&#34;/tenants/acme/databases/orders&#34;) -&gt; Some((&#34;acme&#34;, &#34;orders&#34;))&#xA;&#xA;#### Reserved prefixes&#xA;- extract_tenant_database(&#34;/health&#34;) -&gt; None&#xA;- extract_tenant_database(&#34;/metrics&#34;) -&gt; None&#xA;- extract_tenant_database(&#34;/ui/collections&#34;) -&gt; None&#xA;- extract_tenant_database(&#34;/control/tenants&#34;) -&gt; None&#xA;- extract_tenant_database(&#34;/favicon.ico&#34;) -&gt; None&#xA;&#xA;#### Malformed&#xA;- extract_tenant_database(&#34;/&#34;) -&gt; None&#xA;- extract_tenant_database(&#34;/tenants&#34;) -&gt; None (no tenant segment)&#xA;- extract_tenant_database(&#34;/tenants/&#34;) -&gt; None&#xA;- extract_tenant_database(&#34;/tenants/acme&#34;) -&gt; None (no /databases/ segment)&#xA;- extract_tenant_database(&#34;/tenants/acme/databases&#34;) -&gt; None&#xA;- extract_tenant_database(&#34;/tenants/acme/databases/&#34;) -&gt; None&#xA;- extract_tenant_database(&#34;/tenants/acme/orders&#34;) -&gt;
