    <description>
Credential (JWT) issuance, listing, and revocation endpoints on the control plane. Integrates B1&#39;s JwtIssuer, B4&#39;s role-ceiling enforcement, and C1&#39;s authz pattern.&#xA;&#xA;## Scope&#xA;&#xA;### 1. Routes (extend control_plane_routes.rs)&#xA;&#xA;    .route(&#34;/tenants/:id/credentials&#34;, post(issue_credential))&#xA;    .route(&#34;/tenants/:id/credentials&#34;, get(list_credentials))&#xA;    .route(&#34;/tenants/:id/credentials/:jti&#34;, delete(revoke_credential))&#xA;&#xA;### 2. Handlers&#xA;&#xA;#### issue_credential&#xA;&#xA;Body: { target_user: &#34;&lt;uuid&gt;&#34;, grants: { databases: [...] }, ttl_seconds: u64 }.&#xA;&#xA;Authorization logic:&#xA;- Require the caller is either a deployment admin OR the target_user is the caller themselves (self-issue).&#xA;- If caller is a deployment admin, they can issue to any user in the tenant (subject to target&#39;s role ceiling).&#xA;- If self-issue, the grants must be within the caller&#39;s own TenantRole::delegable_ops ceiling (via TenantRole::enforce_ceiling from B4).&#xA;- Call the same enforce_ceiling for the TARGET user&#39;s role even when the caller is a deployment admin — you cannot issue a credential exceeding the target&#39;s own rights.&#xA;&#xA;Look up target_user&#39;s TenantMember.role via storage.get_tenant_member. If not a member, return 404/403.&#xA;&#xA;Build JwtClaims:&#xA;- iss = the deployment&#39;s issuer string (from ControlPlaneState config or a const like &#34;axon-local&#34;)&#xA;- sub = target_user UUID&#xA;- aud = tenant_id string&#xA;- jti = fresh UUID&#xA;- iat = now seconds&#xA;- nbf = now seconds (no future window for basic issuance)&#xA;- exp = now + ttl_seconds&#xA;- grants = request body grants&#xA;&#xA;Sign with B1&#39;s JwtIssuer using the deployment signing key (pull from ControlPlaneState or a new configurable HMAC key). For this bead, accept a hardcoded dev key via env var AXON_JWT_KEY (fallback to a fixed default for tests). Production key management is a later bead.&#xA;&#xA;Persist issuance metadata (jti, user_id, tenant_id, issued_at, expires_at, revoked=false, grants JSON) via a new storage method track_credential_issuance. The signed JWT is returned ONCE in the response body; never persist the signed string.&#xA;&#xA;Response: { jwt: &#34;eyJ...&#34;, jti: &#34;&lt;uuid&gt;&#34;, expires_at: &lt;seconds&gt; }&#xA;&#xA;#### list_credentials&#xA;&#xA;Require tenant_admin (sees all) OR the authenticated user themselves (sees own only). Return metadata rows — never the signed JWT (not persisted anyway). Schema: [{ jti, user_id, issued_at, expires_at, revoked, grants }].&#xA;&#xA;#### revoke_credential&#xA;&#xA;Require tenant_admin OR the jti&#39;s owner. Insert row into credential_revocations (table from A1). Invalidate the in-memory LRU cache if B1 exposed one; if not, the next request&#39;s SQL check will catch it. Return 204.&#xA;&#xA;### 3. Storage adapter trait methods&#xA;&#xA;    fn track_credential_issuance(&#xA;        &amp;self,&#xA;        jti: Uuid,&#xA;        user_id: UserId,&#xA;        tenant_id: TenantId,&#xA;        issued_at_ms: i64,&#xA;        expires_at_ms: i64,&#xA;        grants_json: &amp;str,&#xA;    ) -&gt; Result&lt;(), AxonError&gt;;&#xA;&#xA;    fn list_credentials(&#xA;        &amp;self,&#xA;        tenant_id: TenantId,&#xA;        user_filter: Option&lt;UserId&gt;,&#xA;    ) -&gt; Result&lt;Vec&lt;CredentialMetadata&gt;, AxonError&gt;;&#xA;&#xA;    fn revoke_credential(&#xA;        &amp;self,&#xA;        jti: Uuid,&#xA;        revoked_by: UserId,&#xA;    ) -&gt; Result&lt;(), AxonError&gt;;&#xA;&#xA;CredentialMetadata shape: { jti, user_id, tenant_id, issued_at_ms, expires_at_ms, revoked, grants_json }.&#xA;&#xA;New table (add to auth_schema.rs):&#xA;&#xA;    CREATE TABLE IF NOT EXISTS credential_issuances (&#xA;        jti            TEXT PRIMARY KEY,&#xA;        user_id        TEXT NOT NULL,&#xA;        tenant_id      TEXT NOT NULL,&#xA;        issued_at_ms   INTEGER NOT NULL,&#xA;        expires_at_ms  INTEGER NOT NULL,&#xA;        grants_json    TEXT NOT NULL&#xA;    )&#xA;&#xA;credential_revocations table exists already from A1.&#xA;&#xA;Implement in all 3 backends.&#xA;&#xA;### 4. Integration tests: crates/axon-server/tests/control_credentials_test.rs (NEW)&#xA;&#xA;- issue_with_deployment_admin — POST with admin JWT, verify response has signed JWT + jti; verify credential_issuances has a row.&#xA;- issue_over_scoped_returns_403 — caller&#39;s role is write; tries to issue admin grants → 403 grants_exceed_role.&#xA;- self_issue_within_ceiling — caller issues to themselves with grants ⊆ their own role; succeeds.&#xA;- list_credentials_admin — admin sees all credentials in tenant.&#xA;- list_credentials_self — non-admin sees only their own.&#xA;- list_never_returns_signed_jwt — assert response does not contain a &#34;jwt&#34; field anywhere.&#xA;- revoke_by_admin — admin revokes a jti; credential_revocations has the row.&#xA;- revoke_by_owner — credential owner revokes their own; succeeds.&#xA;- revoke_by_unrelated_user_returns_403 — different user tries to revoke someone else&#39;s credential; 403.&#xA;- revoked_credential_rejected_on_next_request — issue a credential, revoke it, use it via B1&#39;s verify pipeline, assert the rejection with error_code credential_revoked.&#xA;&#xA;## Context&#xA;&#xA;- B1 landed JwtIssuer and the 8-step verify pipeline — reuse JwtIssuer::issue and the existing signing key plumbing.&#xA;- B4 landed TenantRole::enforce_ceiling and AuthError::GrantsExceedRole — use them verbatim.&#xA;- C1 landed control_plane_authz::{require_deployment_admin, require_tenant_admin} — extend with self-issue rules.&#xA;- A1 landed credential_revocations table.&#xA;- D3/E-phase will later wire the middleware into every data-plane route; for this bead, the revoke flow is validated via a direct call to the B1 pipeline, not a real HTTP request.&#xA;&#xA;## Files to touch&#xA;&#xA;- crates/axon-core/src/auth.rs — add CredentialMetadata struct&#xA;- crates/axon-storage/src/auth_schema.rs — add credential_issuances table&#xA;- crates/axon-storage/src/adapter.rs — 3 new trait methods&#xA;- crates/axon-storage/src/memory.rs, sqlite.rs, postgres.rs — impl&#xA;- crates/axon-server/src/control_plane_routes.rs — add routes + handlers&#xA;- crates/axon-server/src/control_plane_authz.rs — add self-issue helper or extend require_tenant_admin to allow owner&#xA;- crates/axon-server/tests/control_credentials_test.rs — NEW&#xA;&#xA;## Files NOT to touch&#xA;&#xA;- crates/axon-server/src/auth_pipeline.rs — B1 is canonical; only read from it&#xA;- crates/axon-server/src/control_plane.rs — legacy CRUD&#xA;- crates/axon-core/src/auth.rs variants — do NOT add new AuthError variants (grants_exceed_role already exists from B4)&#xA;&#xA;## Verification&#xA;&#xA;- cargo test -p axon-server --test control_credentials_test (must pass all tests)&#xA;- cargo test --workspace green&#xA;- cargo clippy --workspace -- -D warnings clean
