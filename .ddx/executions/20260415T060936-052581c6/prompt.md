    <description>
Concurrency-safe auto-bootstrap for zero-tenant deployments. On first authenticated request, create a default tenant + default database + add the caller as admin. All under ON CONFLICT DO NOTHING so concurrent first-requests converge on a single row.&#xA;&#xA;## Scope&#xA;&#xA;### 1. New module: crates/axon-server/src/bootstrap.rs&#xA;&#xA;    //! Default-tenant auto-bootstrap per ADR-018 §6.&#xA;    //!&#xA;    //! On a fresh deployment with zero tenants, the first authenticated&#xA;    //! request creates:&#xA;    //!   1. A &#34;default&#34; tenant row in the auth_schema tenants table&#xA;    //!   2. A &#34;default&#34; database in tenant_databases&#xA;    //!   3. A tenant_users row adding the caller as admin&#xA;    //!&#xA;    //! Idempotent — runs only when storage.count_tenants() returns 0, and&#xA;    //! concurrency-safe via ON CONFLICT DO NOTHING.&#xA;&#xA;    pub async fn ensure_default_tenant(&#xA;        storage: &amp;dyn StorageAdapter,&#xA;        caller_user_id: UserId,&#xA;    ) -&gt; Result&lt;DefaultTenantHandle, AxonError&gt;;&#xA;&#xA;    pub struct DefaultTenantHandle {&#xA;        pub tenant_id: TenantId,&#xA;        pub database_name: String,&#xA;    }&#xA;&#xA;The caller identity comes from B1 (ResolvedIdentity) for JWT flow or B3 (resolve_tailscale_identity) for Tailscale. In either case, by the time we reach bootstrap, the user row exists.&#xA;&#xA;### 2. Storage adapter: add a fast count + upsert path&#xA;&#xA;    fn count_tenants(&amp;self) -&gt; Result&lt;usize, AxonError&gt; {&#xA;        Ok(0)&#xA;    }&#xA;&#xA;    fn upsert_default_tenant(&#xA;        &amp;self,&#xA;        name: &amp;str,  // always &#34;default&#34;&#xA;    ) -&gt; Result&lt;TenantId, AxonError&gt;;&#xA;&#xA;upsert_default_tenant SQL:&#xA;&#xA;    INSERT INTO tenants (id, name, display_name, created_at_ms, updated_at_ms)&#xA;    VALUES (?, ?, ?, ?, ?)&#xA;    ON CONFLICT (name) DO NOTHING;&#xA;&#xA;    SELECT id FROM tenants WHERE name = ?;&#xA;&#xA;The first call wins with its generated UUID; subsequent callers get the winning UUID back. Wrap in a transaction.&#xA;&#xA;### 3. ensure_default_tenant logic&#xA;&#xA;    1. count_tenants() — if &gt; 0, we&#39;re NOT in bootstrap; return Err(AxonError::NotFound) so caller uses the explicit tenant URL instead.&#xA;    2. upsert_default_tenant(&#34;default&#34;) — get the tenant_id.&#xA;    3. create_tenant_database(tenant_id, &#34;default&#34;) — ignore ON CONFLICT (idempotent).&#xA;    4. upsert_tenant_member(tenant_id, caller_user_id, TenantRole::Admin) — idempotent.&#xA;    5. Return DefaultTenantHandle { tenant_id, database_name: &#34;default&#34; }.&#xA;&#xA;Each step uses an idempotent write, and all callers converge on the same (tenant_id, database_name) pair.&#xA;&#xA;### 4. Tests: crates/axon-server/tests/bootstrap_test.rs (NEW)&#xA;&#xA;#### bootstrap_first_call_creates_everything&#xA;Empty DB → ensure_default_tenant → verify tenants has 1 row, tenant_databases has 1 row, tenant_users has 1 row.&#xA;&#xA;#### bootstrap_second_call_returns_same_handle&#xA;Call twice with different user IDs → first call creates; second call adds the second user as admin without creating a new tenant. Verify tenants still has 1 row, tenant_users now has 2 rows.&#xA;&#xA;#### bootstrap_skips_when_tenants_present&#xA;Seed with an explicit tenant → ensure_default_tenant returns NotFound error (not the default tenant). The middleware caller will then use the explicit URL path.&#xA;&#xA;#### concurrent_bootstrap_converges (INV-020 primitive)&#xA;Spawn N=64 threads calling ensure_default_tenant with 64 distinct user UUIDs on a fresh DB. After join:&#xA;- Exactly 1 tenant row with name=&#34;default&#34;&#xA;- Exactly 1 tenant_databases row&#xA;- 64 tenant_users rows (one per caller), all pointing at the same tenant_id&#xA;- Every caller returned the same tenant_id in their DefaultTenantHandle&#xA;&#xA;Run on memory AND sqlite backends. This is INV-020 at the primitive level; the full axon-sim workload is a separate bead.&#xA;&#xA;#### bootstrap_uses_default_database_name&#xA;Assert DefaultTenantHandle::database_name == &#34;default&#34;.&#xA;&#xA;## Out of scope&#xA;&#xA;- Router wiring: this bead is the primitive + tests. The HTTP middleware/router that DECIDES when to call ensure_default_tenant (based on path prefix or config) is D3&#39;s job after the path-router is finalized.&#xA;- --no-auth integration: E2 handles the no-auth path-based namespace. E1 is the auth-mode bootstrap.&#xA;- Deleting the &#34;default&#34; tenant: out of scope.
